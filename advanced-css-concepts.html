# CSS Box Model

CSS Positioning allows you to change the position from the normal flow of the CSS box model (which consists of the contents, padding, border, and margin respectively). There are also 4 more aspects to the box model: top, bottom, left and right. These represent the outermost boundaries of an HTML element. The box model can be visualized as:

Notice that the innermost rectangle in the box model is referring to the content (i.e. the information that's inside of the start and end tags). The padding refers to the space that separates the content from the potentially visible boundary. This visible boundary is referred to as the border. The margin refers to the spacing that is outside of the visible border. If we apply the following class to the next p element: .box-model-example {
padding: 10px;
border-top: 10px solid green;
border-bottom: 10px solid blue;
border-left: 10px solid indigo;
border-right: 10px solid violet;
margin: 10px;
}

Here you can see how the border surrounds the padding and the content of this p element. Go inside the inspector of your browser in order to see the outermost boundary of the element. You will notice that the margin is colored orange by the browser if you're using Chrome. Feel free to play around with this class to see how the margin relates to the padding, and vice versa. # Working with CSS position properties

Woah! Is that the American flag? Yep, all of this was created using CSS, SVG, and a powerful css feature called Flexbox. This flag is rendered with relative positioning, absolite positioning, and a few features from Flexbox. But before we discuss the flag in depth, let's discuss what each position property value does.

  • static
  • relative
  • absolute
  • fixed
  • sticky

^ Static Positioning

HTML elements have a static position by default. This means that the elements will be positioned in the order as they appear in the document. Elements that have static positioning can not be altered with the top, bottom, left, or right properties.

^ Relative Positioning

HTML elements that are set with relative positioning will initially be positioned as they appear in the document, but they are able to be changed with the top, bottom, left, and right properties. Recall that the top, bottom, left, and right properties refer to the outermost points of an elements box model. That is, these properties are located at the corners of where the margin ends.

^ Absolute Positioning

Here is where things can get tricky. HTML elements that have absolute positioning are positioned in relation to the nearest ancestor that also has a non-static position. If an absolutely positioned element have no ancestor that has a non-static position, the element is positioned in relation to the browser window at the top.

^ Fixed Positioning

This is similar to the absolutely positioned element in that is relative to the browser window, except that the element doesn't move when the user scrolls up and down. If you look into the source code, the element that displays the orange circle on the bottom right is right below this p element.

^ Sticky Positioning

This positions an element with relative positioning until the scroll bar is on the same level. This position is typically used for header styles.

Now that I've explained what each position value does, let's analyze the CSS/SVG/Flexbox flag class by class. Here is the styling for the flag class:

.flag {
position: relative;
width: 100%;
height: 65vh;
margin: 2% 0;
}

The flag class is set to a position of relative, meaning that it has its normal flow, unless we set the top, bottom, left, and right properties. But here, we aren't setting those properties. Why, you ask? It's because we are going to use an absolutely positioned element inside of the flag element, and remember that an absolutely positioned element's top, bottom, left, and right properties are based on its nearest non-statically positioned element. The width is set to 100%, which means it will horizontally span the entirely of its parent element's width. Height is set to 65 vh, which means that it will vertically span 65% of the viewport's height; the viewport refers to the portion of the browser that is visible to the user. Now let's take a look at the stripe class:

.flag-strip {
width: 100%
height: 7.6923%;
}

.flag-strip:nth-child(odd) {
background-color: red;
}

.flag-strip:nth-child(even) {
background-color: white;
}

The top class has a width of 100%, which will span the entire width of the parent's width. The height is set to 7.6923% because that is the result of 1 / 13. Since there are 13 strips in the American flag, the total height of all the 13 strip classes will equal to 100%. The nth-child(odd) pseudoclass applies a red background to the odd numbered children (that is, the 1st, 3rd, 5th, 7th, and so on element will have a red background).

.blue-field {
position: absolute;
top: 0;
left: 0;
z-index: 10000;
width: 40%;
height: 53.8%;
background: blue;
}

Here, we set the top and left properties to zero because we want the blue field element to sit at the top left edge of the element (therefore mimicking the look of the American flag). The z-index is also a property that is only available to non-statically positioned elements, and what it does is set the overlay order of elements. So the higher the z-index value is, the more likely that element will render on top of other elements, which is why the blue field class is over the red and white strips. If we set the z-index to a negative, the blue field will appear beneath the red and white strips. The z-index doesn't have to be 10000, but it is common practice to set z-index values really high values to ensure that unintentioned elements don't override the z-index.

# CSS Flexbox

CSS has a very powerful tool called Flexbox. This allows content to be layed out in the form of rows and columns. To use Flexbox, set the display property to flex. Once you enable Flexbox, you can now use the other flex properties. The properties that this page will use are: flex-flow and justify-content. flex-flow is a shorthand for two flex properties: flex-direction and flex-wrap. Flex-direction determines what direction the items inside of a flex container will render. The default value is row (meaning the items will be rendered left to right); other values are column, reverse-column, reverse-row. The flex-direction property determines what is the "main axis" of a container. Flex-wrap determines if the items need to be wrapped once the end of the container's width is reached. By default, flex-wrap tries to fit all the items on one row. Justify-content aligns items on the main axis. So in our case, the main axis is row, so the justify-content property will apply from left to right. If flex-direction was set to column, then justify-content would apply from top-to-bottom. Now let's dive into our Flexbox classes.

.star-row {
display: -webkit-box;
display: -ms-flexbox;
display: flex;
-webkit-box-orient: horizontal;
-webkit-box-direction: normal;
-ms-flex-flow: row wrap;
flex-flow: row wrap;
height: 11.1111%;
}

Most of the css rules you see here, you don't have to worry about. We will only focus on display:flex, flex-flow: row wrap, and height: 11.1111%. As stated above, setting display with the value of flex allows the browser to use Flexbox and gives us access to the other flex properties. You always want to put the display:flex rule on a container. The flex-flow property sets two flex properties at the same time: the left value sets flex-direction to row, while the right value sets the flex-wrap property to wrap. The default value for flex-wrap is row nowrap. The height is set to 11.1111% because we need the height of the stars to take up 1/9th of the height of the blue field (since there are 9 rows of stars in the American flag). The other properties were generated by a VS Code tool called Live Sass Compiler. This is somewhat outside of the scope of this course, so it you're interested in learning about SASS/SCSS, send me a message through Slack.

.star-row:nth-child(odd) {
justify-content: space-between;
}

.star-row:nth-child(odd) * {
width: 7%;
}

.star-row:nth-child(even) {
justify-content: space-around;
}

.star-row:nth-child(even) * {
width: 7%;
}

The justify-content rule in combination of the nth-child pseudo class, is applying different spacing to mathc the American flag. The odd numbered children have spaces between each item, while the even numbered items have spaces on both sides.
